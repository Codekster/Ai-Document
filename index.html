Chat & Booked

An AI-powered conversational agent engineered to move beyond simple chat. This project demonstrates a robust, production-ready architecture for building AI systems that understand user intent, retrieve factual data, and execute complex, real-world tasks.

About the Project

The goal of "Chat & Booked" was to solve two critical limitations of modern LLMs: their lack of access to private, real-time data and their inability to perform actions. This project architects a solution where the AI is not just a conversational layer, but the intelligent core of a larger system.

By implementing a Retrieval-Augmented Generation (RAG) pipeline and a secure function-calling framework, "Chat & Booked" acts as a blueprint for creating AI agents that are trustworthy, capable, and useful. The architecture is designed to be a scalable model for any application that requires an AI to interact with proprietary data and execute specific business logic.

RAG (Retrieval-Augmented Generation)

To ensure factual accuracy, every user query first undergoes a retrieval step. The query is converted into a vector embedding and used to search a private MongoDB knowledge base. The top 3-5 most relevant text chunks are retrieved and injected into the LLM's context, effectively 'grounding' the model in real-time, domain-specific data to prevent hallucinations.

Tool Use (Structured JSON Generation)

The LLM, now armed with factual context, analyzes the user's intent. Based on a carefully engineered system prompt, it doesn't generate a freeform text response, but rather a structured JSON object that specifies which 'tool' to use (e.g., show_booking_form). This transforms the LLM into a reliable 'reasoning engine' that delegates tasks.

Backend Execution (Secure Processing)

The JSON command from the LLM is received by a secure Django REST Framework endpoint. The backend acts as the system's source of truth‚Äîit validates the request, authenticates the user, and executes the corresponding business logic. The LLM never has direct access to the database or critical functions.

PDF & QR Generation (Tangible Output)

Once the backend logic is successfully executed (e.g., a ticket is created in the MySQL database), the system generates a tangible output. It uses WeasyPrint and qrcode libraries to create a professional PDF ticket with a unique, scannable QR code, which is then served back to the user, completing the action loop.

What I Learned

"AI is an Orchestrator, Not Just a Generator:"

I learned that the most effective AI systems treat the LLM as an intelligent orchestrator. The real engineering challenge isn't just getting the AI to talk, but building the secure, reliable 'tools' for the AI to use. This project taught me to architect systems where the AI reasons and delegates, and the backend validates and executes.

"The Power of Grounding with RAG:"

I learned firsthand why Retrieval-Augmented Generation is critical for enterprise-grade AI. Relying on a model's pre-trained knowledge alone is unreliable. By building a system to retrieve factual context first, I was able to create a much more accurate and trustworthy assistant.

"The Importance of Structured AI Output:"

Forcing the LLM to respond with structured JSON (for tool use) instead of plain text was a game-changer. It makes the system's behavior predictable and robust, eliminating the need for fragile text parsing on the backend.

"Polyglot Persistence is Practical:"

This project solidified my understanding of using the right database for the right job. Trying to force vector embeddings and unstructured chat logs into MySQL would have been inefficient. Using MongoDB for that specific data type made the entire architecture cleaner and more scalable.
